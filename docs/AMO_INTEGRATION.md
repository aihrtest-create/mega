# Интеграция AmoCRM ↔ Конфигуратор Hello Park

## Что это и зачем

Клиент оставляет заявку через Marquiz → лид появляется в AmoCRM. Менеджер решает отправить ему конфигуратор → одним кликом переводит лид на этап «Отправить конфигуратор» → SalesBot автоматически отправляет клиенту персональную ссылку через Wazzup → клиент заполняет конфигуратор → данные о выборах клиента появляются в AmoCRM: примечание с детальным списком + обновляются поля сделки (дата, гости, сумма).

---

## Полная схема работы

```
1. Marquiz → клиент заполняет квиз на сайте
2. Marquiz → автоматически создаёт лид в AmoCRM (имя, телефон, ответы на квиз)
3. Менеджер открывает лид → переводит в этап «Отправить конфигуратор»
4. SalesBot #27 (триггер: переход в этап) → отправляет сообщение в Wazzup:
      «Имя, заполните конфигуратор: https://.../?lead=<amo_lead_id>»
5. Клиент кликает ссылку → открывается React-конфигуратор
6. Конфигуратор дёргает GET /api/leads/<amo_lead_id> → подтягивает имя и телефон из AmoCRM
7. Клиент проходит все шаги (пакет, квест, локация, еда, дата...) → нажимает «Отправить»
8. Конфигуратор → POST /api/leads/<amo_lead_id>/configure с полным состоянием
9. Сервер → добавляет примечание к лиду в AmoCRM (см. пример ниже)
10. Сервер → обновляет поля сделки: Дата праздника, Детей, Взрослых, Сумма, Ссылка на смету
```

## Как отличить Amo-лид от локального

Amo-лиды используют **числовой id** AmoCRM напрямую (`33153603`).  
Локальные лиды (Telegram/Max бот) используют **nanoid** (`AzVBzGKakELv`).

Функция в `server/src/config/amo-config.js`:
```js
export function isAmoLeadId(id) {
  return typeof id === 'string' && /^\d+$/.test(id);
}
```

Сервер автоматически маршрутизирует запросы к нужному источнику.

## Эндпоинты сервера

### `GET /api/leads/:amoLeadId`
Конфигуратор вызывает при загрузке чтобы получить имя и телефон.

- Если id числовой → идёт в AmoCRM, берёт контакт из сделки
- Ответ: `{ success: true, lead: { id, name, phone, source: 'amo' } }`

### `POST /api/leads/:amoLeadId/opened`
Конфигуратор вызывает когда клиент открыл его.

- Добавляет примечание в AmoCRM: «👀 Клиент открыл конфигуратор»

### `POST /api/leads/:amoLeadId/configure`
Конфигуратор вызывает когда клиент нажал «Отправить».

- Формирует текст примечания из всего выбора клиента
- POST `/api/v4/leads/:id/notes` → добавляет примечание
- PATCH `/api/v4/leads/:id` → обновляет поля

### `POST /api/amo/start-config`
Для SalesBot (зарезервирован, не используется в текущей конфигурации).  
Принимает `{ lead_id }`, возвращает `{ url, name }`.

## Настройка SalesBot (если пересоздавать)

1. AmoCRM → Настройки → Чаты и мессенджеры → Список Salesbot → Создать
2. **Триггер:** «Переход в этап» → Мега Заявки → Отправить конфигуратор
3. **Действие:** «Отправить сообщение» → канал **Wazzup**
4. **Текст:**
   ```
   {{contact.name}}, чтобы мы подготовили точную смету на день рождения,
   заполните конфигуратор — это займёт 3 минуты:

   https://aihrtest-create.github.io/planner-park/?lead={{lead.id}}

   После заполнения мы получим вашу заявку и сразу свяжемся 🎂
   ```
5. Сохранить, убедиться что бот **включён**

## Обслуживание

### Когда токен истечёт (август 2026)
1. AmoCRM → amoМаркет → Hello Park Configurator → Ключи и доступы
2. «Сгенерировать токен» → скопировать
3. На VPS: `nano /root/planner-park/server/.env` → обновить `AMO_LONG_TOKEN`
4. `pm2 restart dr-server --update-env`

### Добавить интеграцию для другой воронки (Ривьера, Сочи и т.д.)
1. Добавить этап «Отправить конфигуратор» в нужную воронку:
   - API: `POST /api/v4/leads/pipelines/{pipeline_id}/statuses` или через UI
   - UI: AmoCRM → Настройки → Воронки → нужная воронка → добавить этап
2. Создать SalesBot с триггером на новый этап (см. выше)
3. Код сервера менять **не нужно** — он работает с любым числовым Amo-лид-ID автоматически

### Деплой изменений сервера
```bash
# На VPS:
cd /root/planner-park && git pull origin main && pm2 restart dr-server --update-env && pm2 logs dr-server --lines 10 --nostream
```

### Проверить что интеграция работает
```bash
curl -s -X POST https://194-87-118-33.nip.io/api/amo/start-config \
  -H "Content-Type: application/json" \
  -d '{"lead_id": 33153603}'
# Должен вернуть: {"url":"https://.../?lead=33153603","name":"...","lead_id":"33153603"}
```

## Ограничения и что можно улучшить

| Что | Статус |
|---|---|
| Работает только в Мега (SalesBot) | Легко добавить другие воронки — см. выше |
| Токен действует ~1 год | Нужно перевыпускать вручную |
| `/api/amo/start-config` открыт без авторизации | Добавить `X-Api-Secret` заголовок (5 минут работы) |
| Повторные открытия конфигуратора = повторные примечания «Клиент открыл» | Добавить дедупликацию при необходимости |
| Поле «Пакет» (select, id=1374451) не заполняется | Нужно получить enum-ID вариантов через API и сопоставить |
