# Упрощенная инструкция по деплою проекта на GitLab

Локальный репозиторий успешно перенаправлен на новый адрес GitLab: [hp-tools/mega](https://gitlab.com/hp-tools/mega).

---

## 1. Как теперь делать деплой?

Теперь нет никакой сложной схемы с ветками или разделением. Вы работаете так же, как привыкли:

1. Внесите изменения в код.
2. Сделайте деплой с помощью команды:
   ```bash
   git add .
   git commit -m "Сообщение об изменениях"
   git push origin main
   ```
   *Поскольку `origin` теперь указывает на GitLab, изменения сразу улетают туда.*

3. Когда вы попросите агента «сделать деплой» в чате, он автоматически выполнит эту последовательность команд в ветку `main`.

---

## 2. Настройка автодеплоя из GitLab на сервер

Чтобы код с GitLab автоматически «подтягивался» на ваш сервер и обновлял сайт `https://party.hello-park.io/mega`, настройте автодеплой. 

Создан файл конфигурации `.gitlab-ci.yml` в проекте. Чтобы он заработал:
1. Зайдите в GitLab в раздел **Settings ➔ CI/CD ➔ Variables**.
2. Добавьте две переменные:
   * **`SSH_PRIVATE_KEY`**: Приватный SSH-ключ для доступа к вашему серверу (VPS).
   * **`SERVER_IP`**: IP-адрес вашего сервера.

При пуше в `main` GitLab автоматически подключится к серверу, выполнит `git pull` и перезапустит проект.
